# Input file specifications

## participant_info.csv

Optional participant-level metadata.

Required columns:
- `Participant`

Optional columns:
- `Sex`

Example:

```csv
Participant,Sex
01,F
02,M
```

## Event CSV files

Each event CSV must contain the following columns:

- `latency`
- `type`
- `position`

The scripts use `position` as the event code written into the EEGLAB dataset.

Typical values include:
- `S2` for SPN
- `RESPONSE` for CNV

## electrode_metadata.csv

Optional channel-level metadata.

Required columns:
- `Electrode`

Optional columns:
- `ROI`
- `Hemisphere`

If `ROI` or `Hemisphere` is not provided, the Python analysis script can infer them from electrode names.

Example:

```csv
Electrode,ROI,Hemisphere
F3,Frontal,Left
Fz,Frontal,Midline
F4,Frontal,Right
```

## run_qc.csv

Generated by `make_run_qc_tables.m`.

Common columns include:
- `participant`
- `runID`
- `keep_primary_run`
- `keep_primary_participant`
- `keep_primary`

The Python script uses `keep_primary` when available.

## speech_accuracy.csv

Optional file for exploratory speech-in-noise analyses.

Required columns:
- `Participant`
- `SNR_dB`
- `Accuracy`
- `n_trials`
- `n_correct`

Example:

```csv
Participant,SNR_dB,Accuracy,n_trials,n_correct
P01,0,0.75,40,30
P01,-5,0.65,40,26
```

## anticipatory_electrode_long_noninterp.csv

Generated by `export_anticipatory_long_noninterp.m`.

Required columns:
- `Participant`
- `RunID`
- `Trial`
- `Condition`
- `Component`
- `Electrode`
- `Value_uV`

Optional columns:
- `Sex`

## anticipatory_electrode_long_interp.csv

Generated by `export_anticipatory_long_interp.m`.

This file follows the same structure as the non-interpolated version.
